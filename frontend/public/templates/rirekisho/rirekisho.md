# Smart Resume Markdown Schema (Japanese Version / 日本語版)

このガイドは、日本の「履歴書」用です。

---

## 1. メタデータ (YAML Frontmatter)
```yaml
---
firstName: "太郎"
lastName: "田中"
furiganaFirstName: "タロウ"
furiganaLastName: "タナカ"
position: "シニアソフトウェアエンジニア"
birth: "1990年1月1日"
email: "taro@example.com"
mobile: "080-1234-5678"
address: "東京都渋谷区..."
---
```

## 2. 構成ルール
- **個人情報**: 氏名は `firstName` (名) と `lastName` (姓) に分けてください。ふりがなも同様に `furiganaFirstName` と `furiganaLastName` に分けます。
- **セクション**: `##` を使用（例：`## 職務経歴`, `## 学歴`）。
- **職歴・学歴の各項目**:
  - `### 会社名・学校名`
  - `**役職・学位** | 開始日 - 終了日`
    - *注意: 役職名のみを太字にします。`|` と ` - ` の前後には必ず半角スペースを入れてください。*
- **日付形式**: `YYYY年MM月` または `YYYY/MM` を使用してください。現在在職中の場合は `現在` を使用します。
- **詳細説明**: `- ` を使用して箇条書き。

---

## 📸 ビジョンモデル（Vision Model）活用のコツ
画像（スクリーンショットなど）を GPT-4o や Gemini Pro Vision などの画像認識対応モデルに読み込ませる場合：
- レイアウト情報を活用するように指示してください。「ヘッダー部分から氏名や連絡先を抽出すること」「行の右端にある日付を、左側の役職名と正しく紐付けること」などを伝えると精度が上がります。
- 構造が複雑な履歴書の場合、テキスト抽出（OCR）よりも画像として認識させたほうが、項目間の関係性を正しく維持できることが多いです。
